# 交接清单

## 给游戏侧的入口

运行脚本：

```powershell
.\package_video.ps1 -Video ".\input_videos\your_video.mp4"
```

输出目录：

```text
theme_packs/<theme_id>/
```

游戏侧读取：

```text
manifest.json
package_status.json
analysis_report.json
pipeline_result.json
assets/
frames/
p1_animation_manifest.json
p2_animation_manifest.json
```

`package_status.json` 里 `validation.can_load_in_game=true` 时可以加载。

## 技能语句读取

长按攻击触发吐槽时，优先读取：

```text
manifest.players.p1.skill_taunt.text
manifest.players.p2.skill_taunt.text
```

兜底顺序：

```text
manifest.taunt.player_taunts.<player>.text
manifest.taunt.text
```

`skill_taunt.source` 会标记来源：`dialogue_or_subtitle`、`observed_from_video`、`derived_from_scene`、`onomatopoeia` 或 `fallback_design`。

## 当前内置示例包

```text
theme_packs/anim_video_5f1f2057_real/
theme_packs/anim_p1_ab3ea18b_real/
```

## 安装

首次使用前执行：

```powershell
.\setup.ps1
```

这会安装：

```text
video-pipeline/node_modules/
```

`node_modules/` 不提交到 GitHub；对方拉取仓库后执行 `.\setup.ps1` 即可恢复依赖。

## 注意

- 默认策略会尽量忠实视频角色/IP。
- 如果画图模型因 IP 相似性拒绝，会自动去掉精确角色名但保留视觉特征重试一次。
- 如果明确接受原创化改编，运行时加 `-SafeRemix`。
- 如果只验证结构，不调用模型，运行时加 `-StructureOnly`。
